# Open Source And Boundaries

These repositories are personal projects built as public reference
implementations for discussing better asset discovery products.

- They are not organization-owned projects.
- They will be published as public repositories.
- They will use explicit open source licenses.
- They are meant to be evaluated transparently on product and technical merit.
- The goal is to avoid conflicts of interest while making the work easier to
  inspect, discuss, and learn from.

Suggested short version for the site or talk:

This demo is based on personal projects that will be published as public open
source repositories with explicit licenses. I want the work evaluated in the
open, on its merits, not because it sits inside an org boundary.
